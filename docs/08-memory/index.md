# 第八章 记忆系统 -- 让 Agent 记住你是谁

你跟 ChatGPT 聊了一下午，关掉窗口再回来，它完全不记得你是谁。你跟 Agent 说"我喜欢简洁的代码风格"，下次对话它又从零开始。这就是 LLM 的"金鱼记忆"问题 -- 每次 API 调用都是无状态的，模型本身不会记住任何东西。

**记忆系统就是给 Agent 装上"大脑的硬盘"。** 短期记忆让它在一次对话中保持连贯，长期记忆让它跨越多次对话记住用户偏好和历史经验。好的记忆系统是 Agent 从"工具"进化为"助手"的关键 -- 一个记得你的助手，和一个每次都要重新自我介绍的工具，体验天差地别。

这一章从最基础的对话历史管理讲起，经过摘要压缩、向量记忆检索等中级技巧，最后深入 MemGPT、时间衰减、记忆整合等前沿架构。

## 核心概念速览

| 概念 | 一句话解释 |
|------|-----------|
| 短期记忆 | 当前对话的消息历史，受上下文窗口限制 |
| 长期记忆 | 跨会话持久保存的信息（用户偏好、历史经验等） |
| 滑动窗口 | 只保留最近 N 条消息的简单策略 |
| 摘要压缩 | 用 LLM 把长对话压缩成简短摘要，节省 Token |
| 向量记忆 | 把记忆向量化存储，按语义相关性检索 |
| 实体记忆 | 提取和追踪对话中的关键实体（人名、偏好等） |
| MemGPT | 受操作系统虚拟内存启发的记忆管理架构 |
| 时间衰减 | 越久远的记忆权重越低，模拟人类遗忘曲线 |
