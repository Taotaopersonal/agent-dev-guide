# 术语表

本术语表收录了 Agent 开发中常见的技术术语，按英文字母排序，提供中英对照和简明定义。

::: tip 使用说明
每个术语标注了**首次出现的章节**，方便你回到原文查看详细讲解。
:::

---

## A

### Action
- **中文**: 动作/行动
- **定义**: Agent 在推理后执行的具体操作，如调用工具、发送请求、返回回复。
- **首次出现**: 第 6 章 -- Agent 核心原理

### Agent
- **中文**: 智能体/代理
- **定义**: 一个能感知环境、做出决策并执行行动的自主软件实体。在 LLM 语境中，指 LLM 驱动的、具有工具使用和自主推理能力的程序。
- **首次出现**: 第 6 章 -- Agent 核心原理

### Agent Loop
- **中文**: Agent 循环
- **定义**: Agent 的核心执行机制：接收输入 -> LLM 推理 -> 执行工具 -> 反馈结果 -> 再次推理，直到任务完成。
- **首次出现**: 第 6 章 -- Agent 核心原理

### Agentic RAG
- **中文**: Agent 化 RAG
- **定义**: 将 RAG 检索过程嵌入 Agent 循环中，Agent 可以主动决定何时检索、检索什么、是否需要再次检索。
- **首次出现**: 第 7 章 -- RAG

### Alignment
- **中文**: 对齐
- **定义**: 确保 AI 系统的行为符合人类意图和价值观的技术和研究方向。
- **首次出现**: 第 13 章 -- 安全与对齐

### Attention Mechanism
- **中文**: 注意力机制
- **定义**: Transformer 的核心组件，允许模型在处理序列时关注输入的不同部分，计算各 Token 之间的相关性权重。
- **首次出现**: 第 2 章 -- LLM 基础原理

## C

### Chain-of-Thought (CoT)
- **中文**: 思维链
- **定义**: 一种 Prompt 技术，引导 LLM 逐步展示推理过程，从而提升复杂任务的准确率。
- **首次出现**: 第 3 章 -- Prompt Engineering

### Chunk / Chunking
- **中文**: 分块
- **定义**: 将长文档切分为较小文本片段的过程，是 RAG 流程的关键步骤。
- **首次出现**: 第 7 章 -- RAG

### Computer Use
- **中文**: 计算机使用
- **定义**: AI Agent 通过模拟鼠标点击、键盘输入等方式直接操控计算机桌面环境的能力。
- **首次出现**: 第 16 章 -- 前沿方向

### Context Window
- **中文**: 上下文窗口
- **定义**: LLM 单次能处理的最大 Token 数量。决定了模型能"看到"多少输入和生成多少输出。
- **首次出现**: 第 2 章 -- LLM 基础原理

### Coordinator
- **中文**: 协调器
- **定义**: Multi-Agent 系统中负责任务分配、进度跟踪和结果整合的中枢 Agent。
- **首次出现**: 第 10 章 -- Multi-Agent

## D

### Decoder
- **中文**: 解码器
- **定义**: Transformer 架构的一部分，负责自回归地生成输出序列。GPT、Claude 等模型采用 Decoder-only 架构。
- **首次出现**: 第 2 章 -- LLM 基础原理

## E

### Embedding
- **中文**: 嵌入/向量表示
- **定义**: 将文本（或其他数据）转换为固定维度的数值向量，使语义相似的内容在向量空间中距离相近。
- **首次出现**: 第 7 章 -- RAG

### End Turn
- **中文**: 结束轮次
- **定义**: Anthropic API 中的 stop_reason 值，表示模型认为已完成回答，不需要继续调用工具。
- **首次出现**: 第 5 章 -- Tool Use

### Evaluation
- **中文**: 评估
- **定义**: 对 Agent 系统的输出质量、效率、安全性等维度进行量化测量的过程。
- **首次出现**: 第 11 章 -- 评估与测试

## F

### Few-Shot Prompting
- **中文**: 少样本提示
- **定义**: 在 Prompt 中提供少量示例（通常 2-5 个），引导模型按照期望的格式和风格输出。
- **首次出现**: 第 3 章 -- Prompt Engineering

### Fine-Tuning
- **中文**: 微调
- **定义**: 在预训练模型的基础上，使用特定领域的数据继续训练，使模型在该领域表现更好。
- **首次出现**: 第 2 章 -- LLM 基础原理

### Function Calling
- **中文**: 函数调用
- **定义**: LLM 生成结构化的函数调用请求的能力。OpenAI 术语，对应 Anthropic 的 Tool Use。
- **首次出现**: 第 5 章 -- Tool Use

## G

### Grounding
- **中文**: 接地/事实锚定
- **定义**: 将 LLM 的输出与真实数据源关联，减少幻觉。RAG 是一种典型的 Grounding 技术。
- **首次出现**: 第 7 章 -- RAG

### Guardrails
- **中文**: 护栏/安全边界
- **定义**: 对 Agent 输入输出进行过滤和限制的安全机制，防止有害内容生成或敏感信息泄露。
- **首次出现**: 第 13 章 -- 安全与对齐

## H

### Hallucination
- **中文**: 幻觉
- **定义**: LLM 生成看似合理但实际不正确或虚构的内容。是 LLM 的核心局限之一。
- **首次出现**: 第 2 章 -- LLM 基础原理

### Human-in-the-Loop (HITL)
- **中文**: 人机协同
- **定义**: Agent 在关键决策点暂停，请求人类确认后再继续执行的设计模式。
- **首次出现**: 第 6 章 -- Agent 核心原理

### Hybrid Search
- **中文**: 混合检索
- **定义**: 结合关键词检索（BM25）和向量语义检索的搜索策略，兼顾精确匹配和语义理解。
- **首次出现**: 第 7 章 -- RAG

## I

### Inference
- **中文**: 推理
- **定义**: 将输入传给训练好的模型并获取输出的过程。在 Agent 语境中也指模型的"思考"过程。
- **首次出现**: 第 2 章 -- LLM 基础原理

## J

### JSON Schema
- **中文**: JSON 模式
- **定义**: 用于描述 JSON 数据结构的声明式语言。在 Tool Use 中用于定义工具参数的格式和约束。
- **首次出现**: 第 5 章 -- Tool Use

## K

### Knowledge Base
- **中文**: 知识库
- **定义**: 存储结构化或非结构化知识的数据系统，通过 RAG 技术供 Agent 检索和引用。
- **首次出现**: 第 7 章 -- RAG

## L

### LangChain
- **中文**: LangChain（专有名词）
- **定义**: 最流行的 LLM 应用开发框架，提供链式调用、Agent、RAG 等标准化组件。
- **首次出现**: 第 9 章 -- Agent 框架

### LangGraph
- **中文**: LangGraph（专有名词）
- **定义**: 基于图（Graph）的 Agent 编排框架，用状态机模型管理 Agent 工作流。LangChain 团队开发。
- **首次出现**: 第 9 章 -- Agent 框架

### LLM (Large Language Model)
- **中文**: 大语言模型
- **定义**: 基于 Transformer 架构、在大规模文本数据上预训练的语言模型。如 Claude、GPT-4、Gemini。
- **首次出现**: 第 2 章 -- LLM 基础原理

## M

### MCP (Model Context Protocol)
- **中文**: 模型上下文协议
- **定义**: Anthropic 提出的开放协议标准，用于标准化 AI 模型与外部数据源和工具之间的通信。
- **首次出现**: 第 12 章 -- MCP 协议

### Memory (Agent Memory)
- **中文**: 记忆系统
- **定义**: Agent 存储和检索历史交互信息的机制，包括短期记忆（当前对话）和长期记忆（跨会话持久化）。
- **首次出现**: 第 8 章 -- 记忆系统

### Multi-Agent System
- **中文**: 多 Agent 系统
- **定义**: 由多个具有不同职责的 Agent 组成的协作系统，通过分工和通信完成复杂任务。
- **首次出现**: 第 10 章 -- Multi-Agent

## O

### Observation
- **中文**: 观察
- **定义**: ReAct 模式中，工具执行后返回的结果。Agent 根据观察结果决定下一步行动。
- **首次出现**: 第 6 章 -- Agent 核心原理

### Orchestration
- **中文**: 编排
- **定义**: 协调多个 Agent 或多个步骤按照特定流程执行的过程。LangGraph 是一种编排框架。
- **首次出现**: 第 10 章 -- Multi-Agent

## P

### Planning
- **中文**: 规划
- **定义**: Agent 在执行前先制定步骤计划的设计模式。包含计划生成、执行和动态调整。
- **首次出现**: 第 6 章 -- Agent 核心原理

### Prompt Engineering
- **中文**: 提示工程
- **定义**: 设计和优化输入给 LLM 的提示文本的技术，以获取更好的输出质量。
- **首次出现**: 第 3 章 -- Prompt Engineering

### Prompt Injection
- **中文**: 提示注入
- **定义**: 攻击者通过在输入中嵌入恶意指令来操纵 LLM 行为的安全攻击方式。
- **首次出现**: 第 13 章 -- 安全与对齐

## R

### RAG (Retrieval-Augmented Generation)
- **中文**: 检索增强生成
- **定义**: 先从外部知识库检索相关信息，再将检索结果作为上下文传给 LLM 生成回答的技术。
- **首次出现**: 第 7 章 -- RAG

### ReAct (Reasoning + Acting)
- **中文**: 推理与行动
- **定义**: 一种 Agent 设计模式，交替进行推理（Thought）、行动（Action）和观察（Observation）。
- **首次出现**: 第 6 章 -- Agent 核心原理

### Reflection
- **中文**: 反思/自我反思
- **定义**: Agent 对自身输出进行审视和评估，发现错误后自行修正的设计模式。
- **首次出现**: 第 6 章 -- Agent 核心原理

### Reranking
- **中文**: 重排序
- **定义**: 对初步检索结果进行二次排序，提升最终结果的相关性。常用于 RAG 流程的检索优化。
- **首次出现**: 第 7 章 -- RAG

### Resource (MCP)
- **中文**: 资源（MCP）
- **定义**: MCP 协议中，Server 暴露的可读数据端点。与 Tool 不同，Resource 是被动读取而非主动执行。
- **首次出现**: 第 12 章 -- MCP 协议

### Routing
- **中文**: 路由/分发
- **定义**: 根据输入内容的特征，将请求分发到不同处理逻辑的设计模式。
- **首次出现**: 第 6 章 -- Agent 核心原理

## S

### Sandbox
- **中文**: 沙箱
- **定义**: 隔离的执行环境，限制代码或工具的权限范围，防止意外或恶意操作影响宿主系统。
- **首次出现**: 第 13 章 -- 安全与对齐

### Semantic Search
- **中文**: 语义检索
- **定义**: 基于向量相似度的搜索方法，能理解查询的语义含义而非仅匹配关键词。
- **首次出现**: 第 7 章 -- RAG

### State Graph
- **中文**: 状态图
- **定义**: LangGraph 的核心概念，用有向图表示 Agent 工作流，节点是处理步骤，边是状态转移。
- **首次出现**: 第 9 章 -- Agent 框架

### Stop Reason
- **中文**: 停止原因
- **定义**: LLM API 返回的字段，说明模型为何停止生成。如 "end_turn"、"tool_use"、"max_tokens"。
- **首次出现**: 第 4 章 -- LLM API 实战

### Streaming
- **中文**: 流式输出
- **定义**: LLM 逐 Token 输出结果的方式，无需等待全部生成完毕即可开始接收。
- **首次出现**: 第 4 章 -- LLM API 实战

### System Prompt
- **中文**: 系统提示
- **定义**: 在对话开始前设置的全局指令，定义 AI 的角色、行为规则和能力边界。
- **首次出现**: 第 3 章 -- Prompt Engineering

## T

### Temperature
- **中文**: 温度
- **定义**: 控制 LLM 输出随机性的参数。温度越高，输出越多样；越低，输出越确定。
- **首次出现**: 第 4 章 -- LLM API 实战

### Token
- **中文**: 令牌/词元
- **定义**: LLM 处理文本的最小单位。一个 Token 大约对应 0.75 个英文单词或 0.5 个中文字。
- **首次出现**: 第 2 章 -- LLM 基础原理

### Tool (MCP)
- **中文**: 工具（MCP）
- **定义**: MCP 协议中，Server 暴露的可执行操作端点。模型可以决定何时调用、传什么参数。
- **首次出现**: 第 12 章 -- MCP 协议

### Tool Use
- **中文**: 工具使用
- **定义**: LLM 通过结构化输出请求外部工具执行特定操作的能力。Anthropic 术语，对应 OpenAI 的 Function Calling。
- **首次出现**: 第 5 章 -- Tool Use

### Tracing
- **中文**: 追踪/链路追踪
- **定义**: 记录 Agent 执行过程中每一步操作的详细信息，用于调试、监控和评估。
- **首次出现**: 第 11 章 -- 评估与测试

### Transformer
- **中文**: Transformer
- **定义**: 2017 年 Google 提出的神经网络架构，基于自注意力机制，是所有现代 LLM 的基础。
- **首次出现**: 第 2 章 -- LLM 基础原理

## V

### Vector Database
- **中文**: 向量数据库
- **定义**: 专门用于存储和检索高维向量的数据库。在 RAG 中用于存储文档 Embedding 并支持相似度搜索。
- **首次出现**: 第 7 章 -- RAG

## W

### Workflow
- **中文**: 工作流
- **定义**: Agent 执行任务的预定义步骤序列。与自主决策不同，工作流是确定性的执行路径。
- **首次出现**: 第 6 章 -- Agent 核心原理

## Z

### Zero-Shot Prompting
- **中文**: 零样本提示
- **定义**: 不提供任何示例，仅通过任务描述引导模型完成任务的 Prompt 技术。
- **首次出现**: 第 3 章 -- Prompt Engineering
