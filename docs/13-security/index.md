# 第 13 章 安全与对齐

Agent 系统和传统软件有一个根本不同：它的行为不完全由代码决定，而是由 LLM 的输出驱动。这意味着攻击者不需要找到代码漏洞，只要构造一段文字就可能让你的 Agent "叛变"。

本章从攻击面分析出发，带你建立完整的 Agent 安全防线。我们会从最常见的 Prompt Injection 讲起，理解攻击者如何利用自然语言的模糊性绕过系统指令；然后介绍权限控制、代码沙箱、内容过滤等中间层防御手段；最后深入红队测试、PII 脱敏引擎和隐私合规体系，让你的 Agent 系统能够上线面对真实用户。

安全不是一个功能，而是贯穿 Agent 开发全过程的思维方式。一个没有安全防护的 Agent，就像一个没有门锁的房子——平时看着没问题，出事就是大事。

## 核心安全原则

**1. 纵深防御（Defense in Depth）**

不依赖任何单一防护层。输入检查挡住 80% 的攻击，System Prompt 加固再挡一部分，权限控制限制影响范围，输出过滤防止敏感信息泄露——每一层都可能被突破，但攻击者必须突破所有层才能得逞。

**2. 最小权限原则（Least Privilege）**

Agent 在任何时刻只拥有完成当前任务所需的最少权限。能读不给写，能查单表不给全库，能访问公开数据不给内部系统。

**3. 默认拒绝（Default Deny）**

没有明确授权的操作，一律拒绝。白名单优于黑名单——与其列出所有不能做的事，不如只列出能做的事。

**4. 可审计性（Auditability）**

所有关键操作（工具调用、数据访问、权限变更）都要有日志。出问题时能快速定位谁、在什么时间、做了什么、影响了哪些数据。

## Agent 安全的独特挑战

| 维度 | 传统应用安全 | Agent 安全 |
|------|-------------|-----------|
| 攻击面 | 代码漏洞、网络接口 | 自然语言输入、外部数据源 |
| 行为可预测性 | 高（代码决定行为） | 低（LLM 输出不确定） |
| 权限管理 | 静态权限模型 | 动态工具调用权限 |
| 数据风险 | SQL 注入、XSS | Prompt 注入、知识泄露 |
| 测试方法 | 单元测试、渗透测试 | 红队测试、对抗评估 |

## 你将学到的核心技能

- **入门篇**：识别 Prompt Injection 攻击，实现输入检查 + System Prompt 加固
- **进阶篇**：构建工具权限控制体系，实现代码执行沙箱，部署内容安全过滤
- **高级篇**：建立红队测试流程，实现 PII 脱敏引擎，搭建合规审计系统

::: tip 安全是一个持续的过程
没有任何系统是 100% 安全的。安全工作的目标不是"绝对安全"，而是"让攻击成本高于收益"。每次安全事件都是学习的机会——复盘、修补、加强防线，让系统越来越健壮。
:::
