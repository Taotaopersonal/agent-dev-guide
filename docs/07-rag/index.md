# 第七章 RAG 检索增强生成 -- 让 LLM 拥有你的知识库

LLM 很聪明，但它只知道训练数据里的东西。你公司的内部文档、最新的产品手册、昨天刚发布的技术规范 -- 这些它统统不知道。更麻烦的是，它还可能自信满满地"编造"答案（幻觉问题）。

**RAG（Retrieval-Augmented Generation）就是解决这个问题的核武器。** 核心思路很简单：用户提问时，先从你的知识库里搜出相关内容，然后把这些内容连同问题一起交给 LLM，让它"带着参考资料"回答。就像开卷考试一样，有了参考材料，LLM 的回答既准确又有据可查。

这一章从最基础的向量检索讲起，经过分块策略、混合检索、重排序等优化手段，最后到 Agentic RAG 和 Self-RAG 等前沿架构。

## 核心概念速览

| 概念 | 一句话解释 |
|------|-----------|
| RAG | 检索增强生成，先搜索再回答的技术范式 |
| Embedding | 把文本转换成向量（一串数字），语义相近的文本向量也相近 |
| 向量数据库 | 专门存储和检索向量的数据库（ChromaDB、Pinecone 等） |
| 分块（Chunking） | 把长文档切成适合检索的小段落 |
| 混合检索 | 向量搜索 + 关键词搜索，取长补短 |
| Reranking | 对初步检索结果二次排序，提高相关性 |
| Agentic RAG | Agent 自主决定是否检索、如何检索 |
| Self-RAG | LLM 自我评估检索质量，决定是否需要重新检索 |
