# 第五章 Tool Use -- 让 LLM 长出手脚

大语言模型很聪明，但它只是一个"大脑"：能思考、能推理、能生成文字，却没有手脚。它不能上网查天气，不能执行代码，不能发邮件，甚至不知道现在几点。

**Tool Use（工具使用）就是给 LLM 装上手脚的技术。** 你告诉模型"这里有一些工具可以用"，模型在需要时会生成一段结构化的调用请求（JSON），你的程序负责真正执行，再把结果喂回给模型。模型扮演决策者，程序扮演执行者，两者配合完成任务。

这一章从最基础的工具定义讲起，经过完整的调用流程和多工具协同，最后到动态工具生成和安全沙箱等高级话题。学完之后，你将能够为 LLM 配备任意能力，为后面构建真正的 Agent 打下坚实基础。

## 核心概念速览

| 概念 | 一句话解释 |
|------|-----------|
| Tool Use / Function Calling | 让 LLM 能调用外部函数的机制 |
| JSON Schema | 描述工具参数结构的标准格式 |
| `stop_reason: "tool_use"` | 模型表示"我需要调用工具"的信号 |
| `tool_result` | 你把工具执行结果返回给模型的消息 |
| 并行调用 | 模型一次请求多个独立工具，可同时执行 |
| 链式调用 | 工具 A 的结果作为工具 B 的输入，模型自动编排 |
| 动态工具 | 根据上下文动态生成工具列表，而非固定注册 |
| Human-in-the-Loop | 高风险操作前加入人类确认环节 |
